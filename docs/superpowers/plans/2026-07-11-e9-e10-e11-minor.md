# E9/E10/E11: Minor Compile-Time Fixes Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or superpowers:executing-plans. Steps use checkbox (`- [ ]`).

**Goal:** Fix three minor compile-time issues in `SIRCompiler` (audit findings E9, E10, E11).

**Architecture:**
- **E9** — `ByteString.fromHex(<literal>)` (`SIRCompiler.scala` ~`:1962`) called `fromHex` with no error handling, so an invalid hex literal threw and crashed the plugin ("unhandled exception in the compiler plugin"). Wrap it (and the `hex"…"` interpolator path, which had the same latent crash via the throwing `error` helper) in `report.error` + return an empty `ByteString`, so an invalid literal is a clean positioned error, not a crash.
- **E10** — `throw new Exc(arg, …)` (`compileThrowException`) used the first constructor argument as the error message regardless of type, producing an ill-typed message SIR (`LoweringException: Error message should be String, but got Int`). Only use `arg` when `arg.tpe.widen <:< defn.StringType`; otherwise fall through to the expression text.
- **E11** — the `globalDefs` map (`SIRCompiler.scala:243`) is never written, so `isInGlobalEnv` is always false and two branches of `compileIdentOrQualifiedSelect` (including a `TypeMismatch` consistency check) are dead. Remove `globalDefs`, the dead `CompileDef`/`TopLevelBinding` types, and collapse the 4-case `(isInLocalEnv, isInGlobalEnv)` match into `if isInLocalEnv`.

**Tech Stack:** Scala 3 compiler plugin (dotty API), sbt, ScalaTest.

## Global Constraints

- E9: use `report.error(msg, srcPos)` (non-throwing) — the throwing `error` helper re-throws and escapes as a plugin crash. Return `Constant.ByteString(ByteString.empty)` so types stay consistent (no spurious second error).
- E10: guard the first `compileThrowException` case with `&& arg.tpe.widen <:< defn.StringType`.
- E11: `CompileDef`/`TopLevelBinding` are used only by the dead code; `TypeMismatch` is used elsewhere and stays. Plugin build has `-Wunused` off, so removal is clean.
- `docs/internal/UPLC_CORRECTNESS_AUDIT.md` and `PatternMatchAuditTest.scala` stay untracked. Its 2 failing tests are by-design.
- Commit to `master`, `scalafmtAll` before commit, `git pull --rebase` before push, no `Co-Authored-By` trailer.
